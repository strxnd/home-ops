# Babysitter Project Metadata

This directory contains optional Babysitter configuration for this repository. It is intended for people or agents who use Babysitter across devices or as part of local project orchestration.

Non-Babysitter users can ignore this directory.

## Files to share

These files are safe and useful to keep in Git:

- `project-profile.json` — machine-readable project profile generated by Babysitter.
- `project-profile.md` — human-readable project profile.
- `quality-gates.json` — local quality-gate expectations for Babysitter runs.
- `package.json` and `package-lock.json` — pinned Babysitter SDK dependency for reproducible local use.

## Local-only files

These are intentionally ignored by `.gitignore`:

- `runs/` — local Babysitter run journals and task artifacts.
- `logs/` — local Babysitter logs.
- `inputs/` — local run input files.
- `node_modules/` — local npm install output.
- `creds.env` and `creds.env.tmp.*` — local credentials; never commit these.

## Repository rules still win

`../AGENTS.md` remains the source of truth for this repository. Babysitter users must still follow its safety rules, especially:

- Start with `git status --short`.
- Do not overwrite user changes.
- Do not decrypt, reveal, print, or modify secrets without explicit confirmation.
- Prefer declarative GitOps changes and safe local validation.
- Ask before live cluster, Talos, bootstrap, destructive, commit, push, or reconcile operations.
